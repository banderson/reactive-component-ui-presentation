Why React?: Rendering


  • Virtual DOM : does a diff on every render call and only updates parts of the DOM that have changed

  • Batched updates
    • Entire framework can run in less than one requestAnimationFrame (16ms)
    • Changes are queued up and applied in batches
  • Synthetic Event System : 
    • Object Pooling: no new junk created when mouse across screen
    • Event delegation by default: all events bubble up to one main handler
